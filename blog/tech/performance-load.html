提高性能的几个策略

性能提升

姚川松 写于 2015/7/12

提高HTML的加载速度

首先, 在浏览器解析HTML文档时, 为了提高解析速度, 首先就应该删除掉HTML文件中不必要的空格和注释, 在最终的产品中, 这些内容也是没有意义的.

由于解析HTML文档时, 当通过script或者link标签引入一个外部文件时, 引擎会停下来, 等待文件加载成功, 因为这些文件可能会修改之前创建的DOM树, 所以采用同步的方式加载, 那么减少加载的文件可以减少HTTP数量, 从而提高加载速度. 可以通过合并插件concat实现这一需求. 当遇到img之类的标签时, 会使用同步加载方式. 即不会等待图片加载完成, 会先渲染后面的元素.

减少外部文件的引用, 域名解析也是需要时间的.

CSS查找速度

提高CSS选择器定位到DOM元素的速度, 从而加快渲染速度.

CSS的元素查找器是从右往左的, 因此当采用子元素查找时需要注意, 类似#special-ul li这样的选择器性能比较差, 因为引擎会先去查找所有的li元素, 再看它们有没有id为special-ul的父元素. 所以在使用子元素选择器时, 应该尽量定义好相对关系, 比如直接父子关系, 兄弟关系等, 避免不必要的查找.

尽量使用class和id定位元素, 如果某个组件的子元素关系固定了(比如如下的例子), 不希望引入过多的class, 在使用标签时也应该定义准确, 缩小范围, 减少不必要的查找. 

<div class="container">
    <h2></h2>
    <div><div>
<div>
.container {
    width: 100px;
}
.container>h2 {
    font-weight: 500;
}
.container>h2+div {
    font-weight: 100;
}

通过id是最快的, 其次是class, 后代选择器最慢.
