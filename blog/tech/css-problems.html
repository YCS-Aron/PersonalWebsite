导致浏览器兼容问题的原因

导致CSS兼容问题的主要原因三个: css属性默认值不一致, 比如ul, form元素的padding和margin; css属性的解释不一致, 比如padding在IE和FF下对宽度的影响不一样; css属性的支持不一致, 比如IE6不支持max-height.

而之所以会出现以上三种情况的原因是早期版本的IE还不是标准的浏览器, IE在向标准浏览器逐渐靠拢的过程中, 经历了IE6, IE7, IE8, IE9. 基本上从IE8开始, CSS方面和标准浏览器几乎一致了. 所以兼容问题是由早期版本的IE6, IE7导致的, 兼容也是兼容这两个版本的浏览器.

区分浏览器

区分浏览器主要依靠不同浏览器对css标记的不同支持来实现. 以下是不同浏览器对标记的支持情况:

  • IE6: 支持IE的特有标记*, 同时支持仅仅IE6下才能用的_, 不支持标准浏览器的!important.
  • IE7: 支持IE的特有标记*, 不支持_, 开始支持标准浏览器的!important. 还支持IE7特定的标记+
  • IE8及以上版本的IE和标准浏览器: 不支持*, 不支持_, 支持!important. 也就是说, 从IE8开始, IE抛弃了老的CSS标记, 变得比较标准了.

为了实现兼容, 需要根据不同的浏览器设置样式, 下面是一个在不同浏览器下显示不同高度的例子: 

.test {
    height: 300px;              //IE8和标准浏览器下高度为300px, 因为标准浏览器不支持*, 所以后面两个不能识别
    *height: 100px;             //IE6下高度为100px,因为IE6不支持!important,所以只能识别前两种,而该样式有*,所以优先采用.
    *height: 200px !important;  //IE7下高度为200px,因为只有IE7才同时支持*和!important
}

display:none和visibility:hidden的区别

display:none会让元素不占用页面上的任何空间, 即从DOM树中移除了.

visibility:hidden只会让元素隐藏, 元素依旧占用着之前的页面空间, 影响着页面的布局.

display:inline block inline-block的区别

display:inline设置元素为内联元素, 即没有高宽的属性, 在内联元素上设置高宽是不起作用的. 内联元素的高宽是根据内容的大小的决定的. 而且, 内联元素会逐个排列, 只有当一行显示不下的时候, 才会换行显示.

display:block设置元素为块级元素, 块级元素有高宽的属性, 而且每个块级元素会独占一行. 该行中多余的部分会成为该块级元素的margin.

display:inline-block. 此时, 元素即拥有块级元素高宽的属性, 同时, 也能像内联元素那样逐个排列.

如何clear float

使用了float样式之后, 定义了float的元素的高度将不再影响其父元素的高度(专业术语怎么说的?), 从而使得父元素的高度可能小于子元素的高度. 有多种方法用于清楚浮动.

第一种是在父元素中添加含有clear样式的div元素(为什么 ???). clear值的含义如下:

  • clear: none; 不清除浮动,默认值
  • clear: left; 清除左侧浮动, 此时父元素的高度为子元素中含有float: left的元素的高度的最大值. 即保证父元素能够包裹住含有float: left的子元素.
  • clear: right; 清除右侧浮动, 此时父元素的高度为子元素中含有float: right的元素的高度的最大值. 即保证父元素能够包裹住含有float: right的子元素.
  • clear: both; 清除两侧浮动, 此时父元素的高度为子元素中定义了float的元素的高度的最大值. 即保证父元素能够包裹住所有定义了float的子元素.

第二种方式是通过给父元素设置overflow:hidden, 即可清除浮动. 清除浮动的效果等价于为父元素添加一个含有clear: both的子元素.

第三种方式也是最不灵活的方法, 即给父元素设置固定的高度即可, 使得高度大于子元素的高度.

在IE和Firefox中水平居中的方式

在Firefox中, 让div居中的方式是设置margin-left: auto; margin-right: auto;即可. 而且这种方法必须为div指定宽度.

而在IE中, 这样是不可行的(我的测试结果是可以FF的方法 ????), 需要为夫元素设置text-align: center;.

对于没有宽度内联元素, 用第一种方法是不行的. 第一种方法只适用于有宽度属性的元素. 当要居中的元素是内联元素时, 可以采用第二种方法, 或者将其设置为块级元素.

ul,form元素的padding问题

这是一个不同浏览器对相同元素的默认值不同而导致的兼容问题

在Firefox中, ul元素的padding默认不为0, margin默认为0. 而在IE中, ul元素的padding默认为0, margin默认不为0. 该问题可以提前设置margin: 0px;padding: 0px;解决. 考虑到浏览器会应用后定义的样式, 该样式可以放在CSS文件的顶部.

很多样式文件的第一行就是*{ margin:0; padding:0;}, 就是用来解决标签的padding和margin默认值不一样的问题的.

在IE6中padding不是width和height的一部分

对于下面的样式, IE6和其他浏览器有不同解释.

.box {
    width: 100px;
    height: 100px;
    padding: 10px;
}

IE6中对应的元素会占用120px的高度和宽度, 因为IE6的width不包含padding. 在IE6以上的IE和FF下, 元素占用的高度和宽度是100px, 包含了20px的padding, 所以元素的实际的高度和宽度是80px.

为了保证拥有padding的元素的高度和宽度在不同浏览器下表现一致, 应该兼容IE6, 可以利用*符号. 代码如下: 

.box {
    *width: 100px;     //兼容IE7及以下版本, 只有IE7及以下版本识别*
    *height: 100px;
    //width: 100px !important; //该样式用来兼容IE7以上版本,如果该兼容性问题只出现在IE7及以下版本, 那么这句就不需要了. 但是如果该问题在IE7以上的版本也有出现, 这句就不能省.
    //height: 100px !important;
    width: 100px;           //兼容标准浏览器
    height: 100px;
    padding: 100px;
}

*符号一定要定义在上面, 为什么???

IE6不识别max-width和min-height

很简单, 因为只有IE6支持_标记, 解决方法如下: 

.box {
    max-width: 100px;   //IE7及以上版本的IE才识别该样式
    _width: 100px;      //IE6才识别_, 所以IE6下宽度设置为100px
}
